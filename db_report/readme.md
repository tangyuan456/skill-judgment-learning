# 数据库性能报告 Skill 说明

这个目录是数据库性能报告 Skill 的学生版材料包，用来帮助学生完成“为目标 Skill 构建评估用例集”的作业。学生需要理解这个 Skill 的理想行为，并围绕它设计测试用例、评分规则和自检方案。

这个目标 Skill 的核心场景是：基于数据库性能测试数据生成报告。学生环境不提供内部数据库或内部 API 权限，因此报告数据只能来自本地文件或用户粘贴的数据，不能通过任务标识去查内部库。

## 当前目录结构

```text
db_report/
├── db_report_skill.zip               # 目标 Skill：数据库性能报告生成 Skill
├── test-resource.zip                  # 配套数据：本地测试数据样例
├── readme.md                         # 本说明
└── 参考示例-仅供模板参考/
    ├── ideal_state.md                # 理想态参考
    └── testcases.yaml                # 简化测试用例参考
```

## 文件说明

| 路径 | 说明 |
|------|------|
| `db_report_skill.zip` | 本题的目标 Skill。学生需要围绕这个 Skill 设计评估用例集，重点检验它在数据库性能报告生成任务中的输出质量。 |
| `test-resource.zip` | 配套测试数据包。里面是学生可用于理解 Skill 行为的本地数据样例，例如 sysbench 日志、标准 records JSON 或迭代历史数据等。学生不需要访问内部数据库。 |
| `readme.md` | 当前说明文件，介绍这个目标 Skill 的背景、目录结构、参考文件含义和作业注意事项。 |
| `参考示例-仅供模板参考/ideal_state.md` | 理想态参考示例，展示如何描述数据库性能报告 Skill 的能力边界、意图识别、数据门控、报告质量和红线。学生可以参考它的写法，但需要根据自己的理解完善理想态规范。 |
| `参考示例-仅供模板参考/testcases.yaml` | 测试用例参考示例，展示 YAML 用例的大致结构，包括 `Input`、`ExpectedOutput` 和 `Judge`。这里的用例数量较少，只用于示范，不是最终交付标准。 |

`参考示例-仅供模板参考/` 里的文件只用于说明期望行为和测试方向，不是必须安装的 Skill 代码，也不是可直接提交的完整答案。

## Skill 行为要点

学生应先解压并阅读 `db_report_skill.zip`，理解目标 Skill 的功能、输入要求和输出格式；如需观察数据形态，可解压 `test-resource.zip` 查看本地数据样例。

使用数据库性能报告 Skill 时，输入必须包含可访问的数据来源：

- 本地 `.log` / `.xlsx` / `.json` / `.csv` 文件。
- 用户直接粘贴的标准 records JSON 或原始测试数据 JSON。

如果用户只说“生成某某场景报告”，但没有提供数据文件或 JSON，应先要求补充数据，不能继续生成正式报告。

## 支持的报告类型

- `single`：单次性能测试报告。
- `comparison`：多产品、多版本或多配置性能对比报告。
- `iteration`：版本迭代/历史趋势报告。
- `custom`：围绕用户指定维度的专项分析报告。

## 学生需要产出什么

学生最终应围绕该 Skill 自行设计并提交评估用例集，建议至少包含：

- 理想态规范：说明这个 Skill 在不同输入、数据质量和报告类型下应该怎么表现。
- Rubrics：定义评分维度、权重和扣分项。
- 测试用例集：每个 Skill 不少于 5 条用例，覆盖核心路径和失败路径。
- Meta-Testcase 自检方案：说明如何检查自己编写的测试用例是否覆盖全面、评分清晰、反例有效。

## 注意事项

- 不要引导学生使用内部数据库、内部 API 或任务标识查询数据。
- 所有 TPS/QPS/P95/P99/duration 等数字必须可追溯到用户提供的本地数据。
- 缺少关键字段时要明确说明数据不足，不能估算、编造或硬编码。
- 参考示例中的 `ideal_state.md` 和 `testcases.yaml` 只是写作参考，学生需要自行扩展为完整评估用例集。
