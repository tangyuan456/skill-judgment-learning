# 数据源接入规范

> 本 Skill 统一数据接入出口：本地文件或粘贴 JSON 都转成同一份 `extracted_data.json`（StandardRecords）供分析/渲染使用。

---

## 1. 标准 StandardRecords 结构

```jsonc
{
  "meta": {
    "products": ["Product-A v1.0", "Product-B v1.0"],
    "scenarios": ["oltp_point_select", "oltp_read_only", "oltp_write_only", "oltp_read_write", "oltp_update_index"],
    "concurrencies": [32, 64, 128, 256, 512, 1000],
    "test_env": {},
    "log_meta": [...],
    "source_info": {
      "type": "local_file | local_data | keyword_only",
      "value": "xxx",
      "rows_fetched": 123
    }
  },
  "records": [
    {"product": "...", "scenario": "oltp_point_select", "threads": 128,
     "tps": 12345, "qps": 12345, "p95_ms": 1.23, "p99_ms": 2.34}
  ]
}
```

---

## 2. 数据源类型派发

`scripts/data_source_adapter.py` 根据 `intent.data_source_type` 派发：

| data_source_type | 适配器函数 | 说明 |
|------------------|-----------|------|
| `local_file` (.log) | `adapter_local_log` | 自包含 sysbench 日志解析 |
| `local_file` (.xlsx) | `adapter_local_xlsx` | 按既定 Sheet/表头读取 |
| `local_file` (.json) | `adapter_local_json` | 标准 records 或 JSON 数组 |
| `local_file` (.csv) | `adapter_local_csv` | 表头与 records 字段一致 |
| `local_data` | `adapter_local_rows` | 粘贴的 JSON 性能数据 |
| `keyword_only` | 提示提供文件 | 未检测到数据文件时抛出提示 |

---

## 3. 本地文件格式规范

### 3.1 sysbench .log 日志

标准 sysbench 输出格式。`scripts/log_parser.py` 自动解析，提取：
- 引擎类型、起始时间、DB 端点、数据集、测试时长
- 每个场景每个并发档的 TPS / QPS / P95 / P99

### 3.2 .xlsx Excel 文件

每个 Sheet 对应一个产品，表头为：

| scenario | threads | tps | qps | p95_ms | p99_ms |

### 3.3 .json 数据文件

支持两种格式：
- 标准格式：`{"meta": {...}, "records": [...]}`
- 数组格式：`[{...}, {...}]`（自动适配为 records）

### 3.4 .csv 数据文件

表头需包含：`product, scenario, threads, tps, qps, p95_ms`

---

## 4. 数据质量门控

`data_source_adapter.quality_check()` 要求：

- `records` 长度 > 0
- `tps/qps/p95_ms` 空值率为 0%
- 输出 `concurrencies_by_product_scenario` 供后续展示

---

## 5. 错误码

| 代码 | 含义 |
|------|------|
| E2001 | 本地文件不存在 |
| E2002 | 不识别的扩展名 |
| E2004 | 解析后无有效数据 |
| E2005 | 空值率超标 |
