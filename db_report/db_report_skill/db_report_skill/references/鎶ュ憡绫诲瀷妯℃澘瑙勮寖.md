# 报告类型模板规范

> 本规范定义 4 类报告的章节结构、图表组合、必需内容。

---

## 1. 通用要求（4 类共享）

### 1.1 封面

```
封面字段：
- title:    {报告类型中文名}（如"性能对比报告" / "迭代演进报告"）
- subtitle: {数据源摘要}（如"Product-A v1.0 vs Product-B v1.0"）
- date:     YYYY-MM-DD
- version:  v1.0
```

### 1.2 章节统一要求

- 章节编号：第 1 章 / 1.1 / 1.1.1（最多 3 级）
- 表头浅蓝底色 `#D9E2F3`（docx + html）
- 数字千分位分隔（`{:,.0f}`）
- 空值显示 `-`
- 图表引用：`![alt](charts/xxx.png)` + 紧跟图例
- 洞察框：`> ✅ L1 | 数据来源：xxx\n>\n> 文本`

---

## 2. report_type=single（单次报告）

### 2.1 章节结构

```
第 1 章 测试结论
  1.1 测试结论摘要（1 段）
  1.2 关键性能数字（1 张表：5 场景峰值）
  1.3 核心结论（3~5 条）

第 2 章 测试环境
  2.1 数据库配置
  2.2 硬件环境
  2.3 测试方法（含 5 场景说明）
  2.4 数据质量

第 3 章 性能数据
  3.1 全量数据（1 张表：5 场景 × N 并发档）

第 4 章 性能分析
  4.1 并发扩展性（5 张图 + 1 段总结）
  4.2 P95 延迟稳定性（5 张图 + 1 段总结）

第 5 章 优化建议
  5.1 调优最佳实践
  5.2 测试边界

第 6 章 附录
  附录 A：sysbench 命令
  附录 B：原始数据来源
```

### 2.2 必需图表（≥ 11 张）

| 类型 | 数量 | 命名 |
|------|------|------|
| 5 场景并发-QPS 曲线 | 5 | `single_qps_{scenario}.png` |
| 5 场景并发-P95 曲线 | 5 | `single_p95_{scenario}.png` |
| 峰值汇总条形图 | 1 | `single_peak_summary.png` |

---

## 3. report_type=comparison（对比报告）

### 3.1 章节结构

```
第 1 章 测试结论
  1.1 测试结论摘要
  1.2 关键性能数字（主产品 vs 对标产品）
  1.3 核心结论（4~7 条）

第 2 章 测试环境
  2.1 数据库测试配置（N 个产品并列）
  2.2 硬件环境
  2.3 测试方法
  2.4 数据质量与公平性

第 3 章 性能数据（每产品一节）
  3.1 {产品 1} 全量数据
  3.2 {产品 2} 全量数据
  ...

第 4 章 横向对比分析
  4.1 单产品分析（每产品一节）
  4.2 产品横向对比（5 场景独立 + 性能比值矩阵 + 综合评分）

第 5 章 选型与调优建议
  5.1 选型决策树
  5.2 适用场景矩阵
  5.3 与对标产品的差异
  5.4 调优最佳实践
  5.5 测试边界

第 6 章 附录
```

### 3.2 必需图表（≥ 17 + N 张，N=产品数）

标准图表组合：
- 峰值 QPS 对比 × 5
- 并发-QPS 对比 × 5
- 并发-P95 对比 × 5
- 综合评分雷达 × 1
- 综合天梯图 × 1
- 单产品 5 场景曲线 × N

---

## 4. report_type=iteration（迭代报告）

### 4.1 章节结构

```
第 1 章 演进结论
  1.1 演进摘要（首版 → 末版 累计变化）
  1.2 关键演进数字（5 场景累计变化表）
  1.3 关键发现（持续优化 / 回归点 / 稳定性）

第 2 章 测试环境
  2.1 版本清单（N 个版本一表）
  2.2 硬件环境（应保持一致；不一致需标注）
  2.3 测试方法
  2.4 数据质量

第 3 章 全版本性能数据
  3.1 {版本 1} 全量数据
  3.2 {版本 2} 全量数据
  ...

第 4 章 演进分析
  4.1 5 场景峰值演进
  4.2 5 场景 P95 演进
  4.3 累计变化分析
  4.4 回归点详细分析（如有）

第 5 章 优化建议
  5.1 持续优化方向（基于趋势）
  5.2 回归修复建议（如有回归点）
  5.3 测试边界

第 6 章 附录
```

### 4.2 必需图表（≥ 12 张）

| 类型 | 数量 | 命名 |
|------|------|------|
| 5 场景峰值演进趋势线 | 5 | `iter_trend_qps_{scenario}.png` |
| 5 场景 P95 演进趋势线 | 5 | `iter_trend_p95_{scenario}.png` |
| 累计变化柱状图 | 1 | `iter_cumulative.png` |
| 回归点散点图 | 1 | `iter_regression.png` |

### 4.3 关键结论模板

- ✅ L1：`{首版} → {末版}，{场景} 峰值 QPS {首值} → {末值}（累计 {±X%}）`
- 📊 L2：`{场景} 连续 {N} 个版本峰值递增，平均每版本提升 {X%}`
- 📊 L2：`{版本} 出现回归（{场景} 峰值下降 {X%} > {阈值}%）`

## 5. report_type=custom（客制化报告）

### 5.1 章节结构

```
第 1 章 分析目标
  1.1 用户需求（直接引用 intent.custom_config.extra_questions）
  1.2 关键发现摘要（≤3 条）
  1.3 核心数据点

第 2 章 测试环境与数据范围
  2.1 数据来源
  2.2 数据子集说明（按 focus_dimension 筛选后的样本量）
  2.3 假设前提

第 3 章 深度分析
  3.x （按 focus_dimension 展开，由 templates/custom_report.py 动态生成子节）

第 4 章 结论与建议
  4.1 主要发现（含 L1/L2/L3 分级）
  4.2 不确定性披露
  4.3 后续测试建议

第 5 章 附录
```

### 5.2 必需图表（≥ 3 张）

| 类型 | 数量 |
|------|------|
| focus_dimension vs 主指标主图 | 1 |
| focus_dimension 不同档位对比图 | 1 |
| 5 场景验证图（如适用）| 1~5 |

## 6. report_data 数据结构（4 类共享）

`templates/{type}_report.py::build_*_report_data(extracted, analysis, insights, intent)` 返回：

```python
{
  "cover": {"title":"...","subtitle":"...","date":"...","version":"..."},
  "sections": [
    {
      "id": "ch1",
      "title": "第 1 章 ...",
      "blocks": [...],          // 章节级 blocks
      "subsections": [...]      // 子节
    }
  ]
}
```

`block` 类型：
- `paragraph` — 文本段
- `heading` — 标题（level 1~5）
- `table` — 表格
- `image` — 图片
- `insight` — 洞察框（含 level + source）
- `code` — 代码块
- `bullet` — 列表

---

## 7. 占位符替换强制规则

所有 `{X}` 形式的占位符必须替换为真实数值。F2 核查项会扫描：

```python
re.findall(r'\{[A-Za-z_][A-Za-z0-9_]*\}', report_text)
```

存在残留即失败。
