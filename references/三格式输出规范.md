# 三格式输出规范

> 本规范继承 `tdsql-b-whitepaper/references/报告模板与排版规范.md` 第 3 章的所有要求，并补充 4 类报告的共用规则。

---

## 1. 输出顺序（强制）

```
1. report.md   — source of truth，最先生成
2. report.docx — 独立生成（python-docx 重写一遍，全文微软雅黑 + 表头浅蓝）
3. report.html — 独立生成（拼接 HTML，内联 CSS，含 TOC）
```

**禁止**：
- 用 pandoc 等工具直接 md → docx/html
- 仅生成一个格式后批量转换其他格式

**允许**：三格式生成函数共享相同的 `report_data` 数据结构。

---

## 2. Markdown 规范

- GFM 标准语法
- 标题：`#` (章) / `##` (节) / `###` (小节) / `####` / `#####`
- 表格：管道分隔，表头分隔行
- 图表：`![alt](charts/xxx.png)` + 紧跟 `*图：xxx*`
- 强调：`**加粗**`
- 引用：`>` 用于 insight 框

---

## 3. Word (.docx) 规范

### 3.1 字体

- 默认字体：**微软雅黑**
- 代码字体：**Menlo**
- 通过 `set_doc_default_font()` 设置全局
- 所有文本必须经过 `add_run_with_font()` 写入

### 3.2 表格

- 表头浅蓝底色 `#D9E2F3`
- 偶数行浅背景 `#F8FAFC`
- 表头加粗 + 1F4E79 颜色

### 3.3 图片

- 宽度 `Cm(15.5)`，居中
- 图下方有图例文字（斜体 + 灰色）

### 3.4 洞察框

- 左侧色条：L1=蓝（1971C2）/ L2=橙（E67700）/ L3=红（C92A2A）
- 标签：`✅ L1 确定性结论` / `📊 L2 统计性发现` / `💡 L3 假设性洞察`

---

## 4. HTML 规范

### 4.1 字体 fallback

```css
font-family: 'Microsoft YaHei', 'PingFang SC', 'Heiti TC',
             'Hiragino Sans GB', 'STHeiti', 'Arial Unicode MS', sans-serif;
```

### 4.2 配色

| 元素 | 颜色 |
|------|------|
| h1 | #1f4e79 + 底边框 |
| h2 | #2e75b6 + 左边框 |
| h3 | #4a6d8c |
| 表头 | 渐变 #D9E2F3 → #C5D3EE |
| 表格偶数行 | #F8FAFC |
| insight-l1 | 背景 #E7F5FF + 左边框 #1971C2 |
| insight-l2 | 背景 #FFF9DB + 左边框 #F08C00 |
| insight-l3 | 背景 #FFE3E3 + 左边框 #C92A2A |

### 4.3 必备元素

- `<head>` 含 charset / viewport / title
- 内联 `<style>`（不依赖外部 CSS）
- TOC（生成的目录块）
- `<img>` 路径相对（`charts/xxx.png`）

---

## 5. 渲染器入口

```python
# scripts/render_md.py
def render_md(report_data) -> str

# scripts/render_docx.py
def render_docx(report_data, output_path, charts_abs_dir=None)

# scripts/render_html.py
def render_html(report_data) -> str
```

三个函数都接受同一份 `report_data`，由对应的 `templates/{type}_report.py` 构建。
