# 数据源接入规范

> 本 Skill 统一数据接入出口：无论是本地文件、粘贴 JSON、还是 task_id，最终都转成同一份 `extracted_data.json`（StandardRecords）供分析/渲染使用。

---

## 1. 标准 StandardRecords 结构

```jsonc
{
  "meta": {
    "products": ["TDSQL-B v22.7.2", "TDSQL-B v22.7.3"],
    "scenarios": ["oltp_point_select", "oltp_read_only", "oltp_write_only", "oltp_read_write", "oltp_update_index"],
    "concurrencies": [32, 64, 128, 256, 512, 1000],
    "test_env": {},
    "log_meta": [...],
    "source_info": {
      "type": "task_id | local_file | local_data | keyword_only",
      "value": "xxx",
      "rows_fetched": 123
    }
  },
  "records": [
    {"product": "...", "scenario": "oltp_point_select", "threads": 128,
     "tps": 12345, "qps": 12345, "p95_ms": 1.23, "p99_ms": 2.34,
     "task_id": "...", "plan_id": "...", "report_id": "...", "start_time": "...",
     "raw": {"dimension": {...}, "results_key": "..."}}
  ]
}
```

## 2. 数据源类型派发

`scripts/data_source_adapter.py` 根据 `intent.data_source_type` 派发：

| data_source_type | 适配器函数 | 说明 |
|------------------|-----------|------|
| `local_file` (.log) | `adapter_local_log` | 复用 `tdsql-b-whitepaper/scripts/log_to_excel.parse_log` |
| `local_file` (.xlsx) | `adapter_local_xlsx` | 按既定 Sheet/表头读取 |
| `local_file` (.json) | `adapter_local_json` | 标准 records 或 yunyu 行数组 |
| `local_file` (.csv) | `adapter_local_csv` | 表头与 records 字段一致 |
| `local_data` | `adapter_yunyu_rows` | 粘贴的 yunyu 数据库行 JSON |
| `task_id` / `keyword_only` | **默认走 HTTP API**；设环境变量 `USE_DIRECT_PG=1` 可切到 psycopg2 | 查询后仍经 `adapter_yunyu_rows` 规范化 |

## 3. YunYu HTTP API 数据源（🟢 默认，强烈推荐）

**Skill 已内置连接信息，无需任何额外配置即可使用**。

### 3.1 接口定义

| 项 | 值 |
|----|----|
| URL | `http://yunyu.woa.com/test/api/alarm/sql_query/` |
| 方法 | POST |
| Header | `Content-Type: application/json` + `YUNYU-SYS-ID: check` + `YUNYU-TOKEN: <内置>` |
| Body | `{"p_id": 1, "sql_query": "SELECT ... ;"}` |
| 响应 | `{"code": "Success", "data": {"results": "[{...},{...}]"}}` |

⚠️ **响应解析要点**：`data.results` 是 **字符串形式的 Python repr**（键值用单引号而非 JSON 双引号），必须先 `json.loads` 失败再 `ast.literal_eval` 兜底解析。

### 3.2 配置文件

位置：`config/yunyu_api.yaml`（skill 随包提供默认值）

```yaml
yunyu_api:
  url: http://yunyu.woa.com/test/api/alarm/sql_query/
  sys_id: check
  token: <内置 Token>
  p_id: 1
  timeout: 60
```

支持环境变量覆盖：
- `YUNYU_API_URL` / `YUNYU_API_SYS_ID` / `YUNYU_API_TOKEN` / `YUNYU_API_PID` / `YUNYU_API_TIMEOUT`

### 3.3 关键脚本

- `scripts/db_query_http.py`：构建 SQL → 发 HTTP 请求 → 解析响应
- 入口函数：`execute_via_http(intent)` 返回 `list[dict]`

### 3.4 已知限制（重要）

- **该 Token/表仅包含 CVM 白皮书类性能数据**：`task_id` 形如 `100000430924`（纯数字）或 `878fd4d4_tdsql_intel_x86`（hex+后缀）
- **UUID 格式的 task_id**（如 `75ff5153-e6db-4a6a-83ac-1a7828b1869b`）属于 MrThree / 工作流引擎，不在 `yunyu_test_results` 中
- 查询 UUID 前应先用 MrThree 接口或 YunYu 的 `/test/api/report/detail?taskId=<UUID>` 做存在性校验

### 3.5 MrThree UUID 存在性校验（诊断用）

`scripts/probe_mrthree.py` 可查询 UUID 是否在 YunYu 报告系统中存在：

```
GET http://yunyu.woa.com/test/api/report/detail?p_id=1&taskId=<UUID>&admin=true&formValue={}
```

- 返回 `code=Success + data.jsonUrl`：UUID 存在，可下载原始 JSON
- 返回 `code=ReportFailure.NotFound`：UUID 不存在

## 4. PostgreSQL 直连（🔵 可选，兜底方案）

仅当 HTTP API 不可用、或要查询 HTTP API 之外的数据库时使用。

### 4.1 切换方式

```bash
export USE_DIRECT_PG=1
# 同时提供 config/db.yaml（见 db.yaml.example）
```

### 4.2 配置文件

位置：`config/db.yaml`（用户自行从 `db.yaml.example` 复制并填写）

```yaml
yunyu_test_results:
  host: 9.30.x.x
  port: 5432
  database: yunyu
  user: readonly_user
  password: <CHANGE_ME>
  sslmode: disable
```

支持环境变量覆盖：`YUNYU_DB_HOST` / `YUNYU_DB_PORT` / `YUNYU_DB_NAME` / `YUNYU_DB_USER` / `YUNYU_DB_PASSWORD`。

### 4.3 关键脚本

- `scripts/db_query.py`：构建 SQL（`build_sql`）+ 执行 psycopg2 查询（`execute`）
- HTTP 路径与直连路径**共用** `build_sql()`，确保 SQL 一致

---

## 5. SQL 生成规范（HTTP / 直连共用）

严格来自 dify yml「PostgreSQL查询生成器」节点约束：

1. **❌ 绝对不加双引号**：字段名/表名/JSON 访问键都不加 `"`
   - ✅ `WHERE task_id = 'value'`
   - ❌ `WHERE "task_id" = 'value'`
2. **✅ 字符串值单引号**：`'dd270144_tdsql_intel_x86'`
3. **❌ 不用中文标点**：句号/冒号都用英文
4. **✅ OR 组必须括号包裹**：避免 AND/OR 优先级错误
   ```sql
   WHERE task_id = '...'
     AND (
       (dimension->>'test_name' LIKE '%集中式%' AND dimension->>'test_name' LIKE '%read_only%')
       OR
       (dimension->>'test_name' LIKE '%集中式%' AND dimension->>'test_name' LIKE '%point_select%')
     )
   ```
5. **✅ 多 task_id 用 `IN` 或 `OR`**：不支持数组入参
6. **✅ results_key 用 LIKE**：`results_key LIKE '%threads=1000%'`

---

## 6. 数据质量门控

`data_source_adapter.quality_check()` 要求：

- `records` 长度 > 0（否则 E2004）
- `tps/qps/p95_ms` 空值率为 0%（否则 E2005）
- 输出 `concurrencies_by_product_scenario` 供后续门控②展示

---

## 7. 错误码

| 代码 | 含义 |
|------|------|
| E2001 | 本地文件不存在 |
| E2002 | 不识别的扩展名 |
| E2003 | 数据库配置缺失（仅直连模式） |
| E2004 | 查询返回 0 行（task_id/关键词不存在） |
| E2005 | 空值率超标 |
